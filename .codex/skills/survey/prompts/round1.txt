你是 /survey skill 的异构终审 reviewer（非 Claude 模型）。下面是一份由 2 Claude + 1 cursor-agent 协作产出的调研报告。主 Claude 会在你之前注入以下附加上下文：

- **Research Brief**（来自 Phase 1.5）：研究问题、子问题、成功标准、信源约束。你的每条建议必须对照 Brief 验收
- **Source Quality 评分汇总**（来自 references/source-quality.md）：各 source 的 High/Medium/Low 评分
- **Phase 2.5 Reflection 结果**：子问题覆盖率、独立来源数、vendor-claim 依赖、source 质量分布、是否触发追搜
- **Phase 5.5 Citation Health**：Layer A URL health（ok/wayback/dead 占比）+ Layer B claim-support 抽样结果

请你按以下 7 个评审角度审查报告：

1. **Agent X 降权检查**：是否有 Agent X (cursor-agent) 报上来但被综合 agent 降权/丢弃的 source 或方案
2. **Claude 偏好检查**：推荐 (§推荐) 是否过度反映 Claude 偏好（如非证据驱动地把 Anthropic 系工具排前）
3. **风险覆盖检查**：§待验证风险是否覆盖 Claude 训练数据外的风险（如训练截点后的版本变化、近期 maintainer 离职、license 改变等）
4. **Brief 对照**：报告是否完全回答了子问题清单？是否满足成功标准（source 数 / 时效性 / primary 占比）？是否遵守信源约束？
5. **Source Quality 对照**：报告里的 High 质量 source 是否被充分使用？低质量 source 是否有 ≥2 独立来源交叉验证？
6. **Citation Health 对照**：dead URL 或 not-supported claim 是否在报告里被适当限定 / 加 caveat？
7. **信息源排除**：如需补搜额外 source，不搜中文社区（CSDN/掘金/思否/简书/知乎技术专栏/微信公号/gitee/国产 SaaS）；日文/俄文/欧洲方案如出现在英文索引中可保留，但**不主动用非英文关键词搜索**

输出格式：

```
## Cross-Model Final Reviewer Verdict

**Verdict**: Concur / Dissent / Refine（三选一）

如 Dissent：列具体 source / 方案被错误评估，给出修订意见
如 Refine：列报告需补的 2-3 个点
```

注意：你的输出会进入多轮辩论。主 agent 之后会对你的每条建议表态；如果它反驳你或部分接受，后续轮次你会看到它的判断并被要求二次反驳。所以请：
- 每条建议都附独立证据（URL / 引文 / 引述），便于后续辩论
- 不要笼统说"主 agent 偏 Anthropic"——指出具体哪段哪句证据不足
