/survey # SSOT

针对任意话题的系统性调研，多阶段异构搜索 + 辩论终审

用途

对任意话题做系统性调研——选型比较、最佳实践、社区方案研究。产出结构化 Markdown 报告（含来源评分、置信度标注）+ 音频概要（.m4a）。

Pipeline

flowchart LR P1["Phase 1 问题界定"] --> P15["Phase 1.5 Brief"] P15 --> P2["Phase 2 异构搜索"] P2 --> P25["Phase 2.5 Reflection"] P25 --> P35["Phase 3-5 综合报告"] P35 --> P55["Phase 5.5 引用核查"] P55 --> P6["Phase 6 终审辩论"] P6 --> OUT["报告 md + 音频 m4a"] classDef phase fill:#ecfdf5,stroke:#10b981,stroke-width:2px,color:#064e3b classDef out fill:#fef3c7,stroke:#f59e0b,stroke-width:2px,color:#92400e class P1,P15,P2,P25,P35,P55,P6 phase class OUT out

Phase 2：2 个 Claude agent + 1 个 cursor-agent（GPT）并行异构搜索，对抗 Claude 训练集体盲区。cursor-agent 不可用时自动降级并 banner 提示。

无 flag：/survey 只有一条高质量路径，每个阶段都是质量门禁，不可跳过。需要快速概览请直接问 Claude，不要用 /survey。

触发词

/survey X 调研X方案 比较X和Y X的最佳实践 网上有哪些做X的方式 别人怎么解决X

/debate-review # SSOT

对外部 AI 评审反馈做结构化辩论裁决

用途

收到外部 AI 的评审反馈后，逐条裁决——接受 / 部分接受 / 反驳，形成判断矩阵，并把改动落到原文件。适用于架构 plan 评审、PR review、设计 spec 批注。

输入约束：仅负责 phase 3 裁决。启动前需要 4 份 phase 1/2 文件（arch-verdict、cross-verdict、arch-react、cross-react）全部存在，缺任一则拒绝执行。

触发词

粘贴 AI 评审内容 + "帮我裁决" debate this review 评估这条评审意见

/report-to-html # SSOT

Markdown 长报告 → 交互式单页 HTML

用途

把 200+ 行 Markdown 报告（含表格/流程图/引用链接）转成零构建单文件 HTML——粘性目录、Mermaid 图、Accordion 折叠、Pill 状态徽章、打印友好。Tailwind + Alpine + Mermaid 全 CDN，双击即开，邮件可发。

适用不适用
调研报告、架构文档、方案对比、研究 brief 多文档站（Docusaurus）、PPT 演示（用 pptx skill）、需要后端的应用

触发词

/report-to-html xxx.md 把这个报告转成 HTML 做成交互式网页 生成一个能浏览的 HTML 版本

/conventional-commit # SSOT

基于 git diff 生成规范 Conventional Commits 提交信息

用途

先看实际 git diff --staged，基于真实 diff 写提交信息，不臆测。只输出建议，不自动执行 git commit（除非明确要求）。发现 diff 跨多个无关变更时提示拆分。

输出格式

<type>(<scope>): <简洁祈使句，≤72字符，不加句号>

<body：为什么这样改、影响范围、权衡。每行≤100字符；琐碎改动可省略>

<footer：BREAKING CHANGE: ... / Closes #123，可选>

type ∈ feat | fix | refactor | perf | docs | test | build | ci | chore

触发词

写提交信息 commit 这个 生成 commit message

/project-context # SSOT

项目上下文快速速览，供新任务开始时拉齐上下文

用途

快速输出当前项目的目标、架构概览、关键约束、共享约定、验证方式。约 300 字，信息来源优先仓库内文档，推断部分明确标注。

输出结构

  1. 项目目标 — 解决什么问题，当前阶段/里程碑
  2. 架构概览 — 核心模块、数据流、关键边界
  3. 关键约束 — 性能/兼容/合规/不可改动的部分
  4. 共享约定 — 命名、错误处理、提交规范
  5. 验证方式 — 怎么本地验证一个改动是对的

触发词

介绍一下这个项目 会话开始帮我了解项目 新接手任务时 /project-context

/skill-creator # SSOT Tier1 新增

创建 / 改进 / 评测 skill，支持定量 eval 和触发准确度优化

用途

标准化的 skill 生命周期管理：草稿 → 测试 → 定量评估 → 迭代改进。主要服务于 AICAP SSOT 自身维护——新增 skill 或改进现有 skill 的触发准确度。

能力

  • 从零创建符合 SKILL.md 规范的新 skill
  • 修改/优化现有 skill 的描述和触发词
  • 运行 eval 测量 skill 性能（含方差分析）
  • Benchmark 多版本 skill，给出改进建议

触发词

/fedex-tracker # 本地

批量查 FedEx 发货日期，输出 CSV，绕过 Akamai 反爬

用途

用真实 Chrome（CDP 驱动）模拟点击 FedEx 官网追踪流程，提取发货日期（Picked-up date），批量输出 CSV。随机延迟 8-15s 绕过 Akamai Bot Manager。

工作流程（展开查看）
  1. 打开 FedEx Tracking 导航下拉（真实点击，不走 URL 跳转）
  2. 在可见输入框键入追踪号
  3. 点击橙色 TRACK 按钮，等待结果页渲染
  4. 点击 "View more details" 展开 Shipment Facts
  5. 读取 SHIP DATE（或 Picked up 事件日期作 fallback）

触发词

有一批 FedEx 追踪号需要发货日期 批量查 FedEx 物流
