/survey #SSOT

针对任意话题的系统性调研，多阶段异构搜索 + 辩论终审

用途

选型比较、最佳实践、社区方案研究。产出结构化 Markdown 报告 + .m4a 音频概要。

Phase 1 问题界定 → 1.5 Brief → Phase 2（2 Claude + 1 cursor-agent 并行异构）→ 2.5 Reflection → 3-5 综合 → 5.5 引用核查 → Phase 6 终审辩论 → Finalize

无 flag——每个阶段都是质量门禁，不可跳过。cursor-agent 不可用时自动降级并 banner 提示。
/survey X调研X方案比较X和YX的最佳实践

仅负责 phase 3 裁决。需 4 份 phase 1/2 文件，缺失则拒绝执行。

/aicap-commit #SSOT新

AICAP 专用提交流程，自动更新文档并生成 HTML

用途

检测 .rulesync/skills/ 变动 → 更新 SKILLS.md → 重新生成 SKILLS.html / README.html → pnpm run ai:generate → 规范 commit。提交后询问是否 push，不自动推送。

自动执行步骤：

  1. 检查工作区状态
  2. 若 .rulesync/ 有变动 → pnpm run ai:generate
  3. 若 skills 有变动 → 更新 SKILLS.md
  4. 生成 SKILLS.html
  5. 生成 README.html
  6. 暂存所有生成产物
  7. 生成 Conventional Commit 信息并提交
  8. 询问是否 push
/aicap-commit提交到 AICAP帮我提交这次改动

/sync-aicap #SSOT新

双向同步：拉取最新远端 skills，或把本地新 skill 推给团队（PR）

用途

自动判断模式：若无本地改动则拉取远端最新（git pull → ai:generate → setup:skills）并展示新增/删除/更新 skill diff；若有本地新 skill，则更新文档、创建 feature branch 并开 PR 推给团队。

两种模式：

模式 A：拉取同步
  1. 记录当前 skill 快照
  2. git pull
  3. pnpm run ai:generate
  4. pnpm run setup:skills
  5. 展示新增 / 删除 / 更新 diff
模式 B：推送 PR
  1. 确认本地新 skill 列表
  2. 更新 SKILLS.md / HTML
  3. pnpm run ai:generate
  4. 提交到 feature branch
  5. gh pr create 并返回 PR URL
同步 AICAPsync aicap更新 AICAP把本地 skill 提交 PR
